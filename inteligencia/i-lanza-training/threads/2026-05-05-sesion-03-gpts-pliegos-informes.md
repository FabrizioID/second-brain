## Objetivo del hilo
Registrar lo realizado en la Sesion 03 del programa de capacitacion IA GEN+ para i-Lanza, segun el acta `acta_sesion3_genplus (1).pdf`, y actualizar el contexto del proyecto con ejercicios, aprendizajes, bloqueos y acuerdos.

## Resumen ejecutivo
- La Sesion 03 paso de productividad con herramientas sueltas a construccion de flujos con GPTs personalizados: actas, revision de pliegos/TDR e informes de ingenieria.
- Se reforzo el marco de trabajo: input -> IA -> estructura -> entregable, con validacion humana como condicion final.
- Quedaron acuerdos concretos para la proxima sesion del jueves 07 de mayo de 2026 a las 15:00 h de Espana.

## Estado del proyecto en este momento
- La Sesion 02 ya no es el ultimo hito operativo; la Sesion 03 fue ejecutada el 05 de mayo de 2026.
- El entrenamiento amplio el foco desde NotebookLM, Gemini/Gmail y Workspace hacia GPTs personalizados en ChatGPT Plus.
- La audiencia ya vio tres ejercicios aplicados a trabajo real: actas, pliegos de licitacion y generacion de informes.
- Sigue activa la necesidad de separar borrador asistido por IA de documento final validado por criterio humano.

## Avances detectados
- Se repasaron LLMs y usos diferenciales: ChatGPT como inteligencia general, Claude para analisis/razonamiento complejo, Gemini para imagenes e integracion Google Drive/Gmail, y NotebookLM para fuente cerrada y menor alucinacion.
- Se presento el flujo clave de trabajo con IA: input -> IA -> estructura -> entregable.
- Se creo un GPT personalizado en ChatGPT Plus con System Instructions, temperatura 0.3 y plantilla corporativa como base de conocimiento.
- El GPT personalizado transformo una transcripcion cruda en acta estructurada con participantes, fecha, plataforma, resumen y temas.
- Se probo exportacion HTML -> navegador/Ctrl+P -> PDF como ruta practica de entrega.
- Se ejecuto ejercicio de revisor automatico de pliegos/TDR sobre un pliego de 136 paginas.
- El GPT con thinking activo genero resumen ejecutivo con citas y numero de pagina por dato, advertencias con nivel de riesgo e interpretacion, y dashboard HTML con indice estimado de riesgo documental del 68%.
- Se ejecuto ejercicio de generador de informes de ingenieria con interprete de codigo activado, usando memorias de calculo y plantilla PDF.
- Se genero informe HTML con encabezados, paginacion y estructura alineada a plantilla, exportable a PDF desde navegador.
- Se hizo dinamica final Kahoot facilitada por Ivana para reforzar conceptos.

## Problemas o bloqueos
- Retay AI quedo bloqueado por politicas TI corporativas de Outlook.
- Para integracion corporativa con Outlook Calendar, falta consultar a Virginia/TI sobre habilitacion de accesos.
- Limite operativo recordado: GPTs personalizados aceptan 20 archivos y 8,000 caracteres en system instructions.
- Riesgo recurrente: que el informe generado sea percibido como final cuando debe mantenerse como borrador validable.

## Decisiones tomadas
- Usar GPT personalizado como mecanismo practico para convertir transcripciones en actas estructuradas bajo plantilla corporativa.
- Para TDR/pliegos, priorizar salida analitica: alcance, riesgos, supuestos y consultas, en vez de resumen narrativo.
- Mantener validacion humana como regla para informes de ingenieria generados con IA.
- Usar HTML -> PDF como ruta rapida de exportacion para entregables generados durante la capacitacion.

## Soluciones o propuestas
- Crear y configurar un GPT personalizado propio para generacion de actas adaptado a plantilla corporativa de cada participante.
- Compartir capturas del dashboard visual generado desde analisis de pliegos en el grupo de WhatsApp.
- Consultar a Virginia/TI sobre habilitacion de Retay AI con Outlook Calendar.
- Reutilizar el patron de dashboard de riesgo documental para futuras sesiones o laboratorios por area.

## Patrones detectados
- La adopcion aumenta cuando el ejercicio termina en un artefacto visible y compartible: acta PDF, dashboard HTML, informe PDF.
- El valor pedagogico esta en mostrar el pipeline completo, no solo prompts aislados.
- Los participantes necesitan distinguir instrucciones de comportamiento/formato frente a archivos de conocimiento/memoria.
- Las restricciones corporativas de TI pueden bloquear herramientas aunque el caso de uso sea claro.

## Oportunidades detectadas
- Convertir el GPT de actas en plantilla replicable para reuniones internas, comites de proyecto y sesiones comerciales.
- Convertir el revisor de pliegos/TDR en piloto de alto valor para Comercial, Legal/Contratos y Oficina Tecnica.
- Convertir el generador de informes en flujo controlado de borradores tecnicos con plantilla y revision humana.
- Usar el dashboard de riesgo documental como pieza demostrativa para direccion: muestra valor inmediato y reduce abstraccion.

## Aprendizajes reutilizables
- En capacitaciones corporativas, el flujo input -> IA -> estructura -> entregable es mas memorable que explicar modelos por separado.
- Para GPTs personalizados, las system instructions definen comportamiento, reglas y formato; los archivos de conocimiento son la memoria.
- En pliegos/TDR, el mejor output no es narrativo sino accionable: alcance, riesgos, supuestos, consultas y evidencia con pagina.
- En informes tecnicos, IA acelera borradores y estructura, pero la responsabilidad tecnica sigue en revision humana.
- La exportacion HTML -> PDF es una ruta suficientemente practica para demos, aunque no reemplaza un sistema documental corporativo final.
